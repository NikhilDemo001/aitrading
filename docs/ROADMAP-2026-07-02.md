## P1 — live-safety, do before real capital

1. **`uvicorn reload=True` in production** (`main.py:3968`): any file edit (or dir deletion — observed today) restarts the process mid-session → in-memory state resets, WS clients drop, bot halts silently. Make it env-driven: reload only when `BOT_DEV=1`.
2. **Weekly-drawdown gate dead in orchestrator** (`orchestrator.py` tick: `weekly_pnl=0.0` hardcoded). Derive week-to-date realized P&L from `history.load_kpi_daily()` and pass it through.
3. **Startup position reconciliation (live)**: `active_positions.json` is trusted blindly on restart. In live mode, reconcile against the broker's positions API at startup; alert on mismatch.
4. **Unify the paper/live flag**: `main.py` speaks `paper_trading`, orchestrator speaks `mode`. This duality *caused* today's P0. Either standardize on one key or make `RiskManager` normalize both at construction.

## P2 — robustness

5. **Atomic state writes**: `active_positions.json` / `trade_history.json` are plain `json.dump` — a crash mid-write corrupts state. Write-to-temp + `os.replace`, or move to SQLite (an `ai_research.db` already exists).
6. **Partial fills**: the execution layer treats every order as fully filled at one price. Live market orders can partial-fill; track `filled_qty` vs `quantity`.
7. **Windows console encoding**: `₹` in log/reason strings raises `UnicodeEncodeError` on cp1252 consoles (hit during today's diagnostics). Set `PYTHONIOENCODING=utf-8` in the launch path or keep log strings ASCII (`Rs.`).
8. **CI**: git now exists — add a remote + a workflow running `pytest -q` and `npm run build` on push. The kill-switch bug survived a day because nothing ran the suite automatically.
