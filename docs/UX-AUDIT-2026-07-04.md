# UX & Accessibility Audit — CIPHER · TERMINAL — 2026-07-04

Evidence-based audit of the live dashboard: computed WCAG 2.1 contrast ratios for every token
pairing in use, WAI-ARIA APG pattern compliance, and interaction-design heuristics for an
operational (money-moving) cockpit.

## Fixed in this pass

1. **`--ink-faint` → `#78829A`** (token, cascades everywhere). Hierarchy preserved — still
   clearly subordinate to `--ink-dim` (5.2 vs 8.1).
2. **Kill Switch text → dark-on-red** (5.61:1). Dark text on red also reads *more* like a
   physical emergency control.
3. **Kill Switch alarm fatigue**: the perpetual pulse desensitizes within an hour of screen
   time (classic alarm-fatigue failure). Now: steady glow at rest, pulse on hover/focus —
   the pulse means "you are about to trigger it," which is what a pulse should mean.
4. **Tabs now implement the WAI-ARIA APG pattern**: roving `tabIndex`, ArrowLeft/Right +
   Home/End navigation, `aria-controls` → `role="tabpanel"` on `<main>` with
   `aria-labelledby` back-reference. Previously `role="tab"` promised keyboard behavior it
   didn't deliver.
5. **Sortable table headers are real `<button>`s** with `aria-sort` on the `<th>` (clickable
   `<th>` alone is invisible to keyboard and screen-reader users), Historical Trades table.
6. **Unlabeled inputs** (watchlist filter, trade search) got `aria-label`s.

## Verified already-good (no action)

- `:focus-visible` ring, token-based and consistent, keyboard-only (no mouse-click rings).
- `prefers-reduced-motion` disables entrances, pulses, tilt, backdrop drift globally.
- P&L never color-only: always signed (`+`/`−`) plus color — colorblind-safe.
- Numerics use `tabular-nums` monospace (no column jitter on tick).
- Empty/loading/error/unloaded states all rendered distinctly (Historical Trades is the model).
- LED strip: visible text labels beside every dot (not color/title-only); `role="status"`.
- Touch: cursor-tilt disabled on coarse pointers; buttons ≥ ~32px effective height.

## Remaining findings (prioritized)

- **P2 — "Why it skipped" tooltips are `title`-only** (ScannerMatrix): invisible to keyboard
  and touch users, and screen readers won't announce them. The information IS critical to the
  product's promise. Proper fix: a focusable disclosure (click/focus-triggered popover) per
  row. Medium effort — needs a small Tooltip/Popover primitive in the design system.
- **P2 — native `confirm()` for Square Off / Kill Switch**: functional and keyboard-accessible
  (that's why it was kept), but it can't communicate severity visually, can't show position
  count/current P&L at the moment of decision, and browsers can suppress repeat dialogs. A
  design-system `ConfirmDialog` with the kill treatment would make the decision informed, not
  just confirmed.
- **P2 — charts have no non-visual alternative**: lightweight-charts canvas is a black box to
  AT. Cheap 80%: an `aria-label` summarizing last price/day range on the chart container,
  refreshed with the data.
- **P3 — sub-tab rows** (Research Lab, Analytics) don't implement the APG tabs pattern either
  (plain buttons — honest, since they don't claim `role="tab"`; upgrading them is consistency
  polish, same recipe as TopNav).
- **P3 — `aria-live` consideration**: ticking P&L should NOT be a live region (it would spam
  screen readers every second). Correct as-is; documenting the intentional decision.
- **P3 — heat-grid tiles** rely on glow intensity for confluence strength; the `n/N` text is
  present on each tile, so data isn't lost — but the glow encoding is sub-threshold for
  low-vision users. Acceptable; note only.

## Heuristic notes (Nielsen) — cockpit-specific

- **Visibility of system status**: excellent (LED strip, status bar, scanner cadence).
- **Error prevention**: kill/square-off confirmations exist; settings form validates via
  schema. The P2 ConfirmDialog upgrade would move from "confirmed" to "informed consent."
- **Recognition over recall**: decision stream + why-skipped tooltips surface the bot's
  reasoning in place. The P2 tooltip fix extends this to all input modalities.
- **Aesthetic ≠ minimalist tension**: data density is the product here; the ambient/motion
  layer stays under the data (faint orbs, transform-only) — correct priority ordering.
