Line-level audit of all backend modules (~19.5k lines) + carried frontend items.
**Supersedes `ROADMAP-2026-07-02.md`** (one correction: its "atomic state writes" item was

## Overall verdict

The codebase is in better shape than its size suggests: no bare excepts in money paths, every
HTTP call has a timeout, no SQL string-building, atomic persistence, duplicate-exit guards,
exit-failure retention, per-symbol isolation in `square_off_all`, spend-gated LLM calls,
dimension-guarded RL policy loads, and test-isolated history paths. The bugs that existed were
*wiring* bugs — features that looked alive but were never invoked correctly.

## P1 — before live capital

1. **Fill-price fidelity** (`upstox_client.py:569-577`, affects `execute_exit` P&L too): live
   MARKET fills are recorded at *LTP at submission*, not the broker's actual average fill.
   Every live P&L number inherits this drift. Fix: after placement, poll the order history
   (`get_order_status` → `average_price`) briefly and record the real fill.
2. **`get_order_status` reads `history[0]`** (`upstox_client.py:707-713`): assumes Upstox
   returns order events newest-first. If oldest-first, this always reports the initial
   "received" state — SL-trigger detection in main.py would never see FILLED. Verify against
   a live response and pin with a test.
3. **Startup position reconciliation (live)**: `load_state()` trusts SQLite/JSON. On live
   restart, fetch broker positions and reconcile; alert on mismatch (manual exits made outside
   the bot, partial fills, etc.).
4. **`_delivery_symbols` is in-memory only** (`upstox_client.py:51,560`): the "this scrip is
   delivery-only" discovery is lost on restart, so live re-attempts a doomed MIS order every
   day. Persist it (config or a small json).
5. **Token-expiry handling**: `_token_expired` pads base64 wrongly when `len%4==0`
   (`'=' * (4 - 0)` = 4 pads → decode error → "not expired") and returns False on any parse
   failure (`upstox_client.py:596-602`). Use `(-len(p)) % 4` padding. Live UX: surface "token
   expires at HH:MM" in the dashboard instead of discovering it when orders fail.

## P2 — robustness

6. **Paper fill fabricates ₹100** when no quote and no price (`upstox_client.py:506`) —
   poisons paper P&L/learning data; reject the fill instead.
7. **`RateLimiter.wait` sleeps holding the lock** (`upstox_client.py:19-28`) — all threads
   serialize behind one sleeper; move the sleep outside the lock.
8. **`save_config` writes the access token into config.json** (`upstox_client.py:111-116`)
   despite its own comment saying it doesn't. Strip credential keys when env supplies them —
   the JWT shouldn't live in a plaintext settings file (it's gitignored, but still).
9. **Live branch of `try_refresh_token` is a no-op** (re-reads config hoping a cron updated
   it). Since Upstox has no refresh grant, make expiry loud: dashboard banner + log with the
   exact expiry time (pairs with P1.5).
10. **Orchestrator uses local `datetime.now()`** while main.py uses IST (`get_ist_now`).
    Correct while the host TZ is IST; add an IST helper to the orchestrator for portability.
11. **CI**: git exists now — add a workflow running `pytest -q` + `npm run build` on push.
    Today's two wiring bugs would both have been caught at introduction by a suite run.
12. **Windows console encoding**: `₹` in logs crashes cp1252 consoles
    (`UnicodeEncodeError` observed during diagnostics). Set `PYTHONIOENCODING=utf-8` at launch
    or keep log strings ASCII.
13. **Redundant weekly filter clause** (`main.py:405-408`): `startswith(today) or >=
    seven_days_ago` — first clause is subsumed; harmless, simplify when touched.
